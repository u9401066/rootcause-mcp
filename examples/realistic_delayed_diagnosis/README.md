# Realistic Delayed Diagnosis Case

> 🔬 **擬真化測試案例** - 模擬真實醫院 HIS 資料的雜訊與挑戰

## 案例概述

**案例類型**: 延遲診斷 (Delayed Diagnosis)  
**結果**: 肺癌疑似病例延誤約 44 天才被發現  
**來源**: 模擬資料 (基於真實情境設計)

## 資料特色 (擬真化處理)

此資料集包含真實 HIS 系統常見的「雜訊」:

| 雜訊類型 | 範例 |
|----------|------|
| 格式跑版 | HL7 報告格式、CSV 欄位 |
| 無關行政事項 | 訂飲料、停車場公告、冷氣抱怨 |
| 錯字 (Typos) | "Breating" (Breathing)、"thout" (thought) |
| 縮寫不一致 | "susp." vs "Suspicious"、"Rt" vs "Right" |
| 關鍵證據混雜 | 重要事件藏在瑣碎紀錄中 |

## 資料來源清單

| 檔案 | 類型 | 描述 |
|------|------|------|
| `DATA_SOURCE_01_EMR_OPD_VISIT.txt` | 門診病歷 | 初次就診紀錄 (2024/01/05) |
| `DATA_SOURCE_02_RAD_REPORT.hl7` | 影像報告 | CT 發現肺部腫塊 (2024/01/07) |
| `DATA_SOURCE_03_NURSING_LOGBOOK.csv` | 護理站日誌 | 包含關鍵事件時間線 |
| `DATA_SOURCE_04_ADMIN_MEMO.msg` | 行政公告 | 可能的干擾因素 |
| `DATA_SOURCE_05_ER_TRIAGE_NOTE.txt` | 急診紀錄 | 事件爆發點 (2024/02/20) |

## 時間線摘要

```
2024/01/05  門診就診 - 醫師安排 CT 檢查
2024/01/07  CT 完成 - 發現 RUL 2.5cm 腫塊 (Critical Finding)
2024/01/07  14:01 報告傳真至護理站
2024/01/07  14:10 護理師將報告放在醫師桌上
2024/01/07  14:30 醫師早退參加研討會
2024/01/07  14:45 清潔人員將文件移至「待歸檔」區
2024/01/07 - 2024/02/20  ??? (44 天空白)
2024/02/20  病人咳血來急診 - 發現報告從未處理
```

## RCA 測試目標

Agent 需要:
1. **從雜訊中提取關鍵事件** - 過濾訂飲料、停車場等無關資訊
2. **重建時間線** - 識別報告傳遞的斷點
3. **識別 Swiss Cheese 漏洞** - 多重防線失效
4. **追溯根本原因** - 為何系統允許此失誤發生?

## 預期挑戰

- 資料分散在 5 個不同格式的檔案
- 關鍵證據 (14:30 醫師早退、14:45 文件被移走) 藏在日常紀錄中
- 病人的假設 ("Dr never called me so I thout it was normal") 揭示溝通斷裂

*此案例不提供標準答案，用於測試 Agent 的推理能力*
